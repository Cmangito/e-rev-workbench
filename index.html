Environmental Review Workflow Prototype

What it is

A local workbench that reads environmental result tables, maps fields, screens QA items, compares starter criteria, and exports auditable review packages.

Consulting value

It reduces repetitive spreadsheet review, standardizes early QA checks, and lets consultants spend more time interpreting results.

AI positioning

The app owns the math. AI is used for drafting support and language review from verified E-Rev findings, not for inventing conclusions.

Next build path

Secure AI drafting, verified criteria libraries, Word/PDF exports, project storage, client templates, and deeper GIS integration.

Project story

E-Rev started as a practical response to a real consulting workflow question: where can AI help without hiding the math? The design keeps screening, trend, profile, QA, and map logic deterministic, then uses AI only after verified findings exist.

Data guide

Using E-Rev Workbench

Review Flow

  1. Upload a CSV, TSV, or Excel file.
  2. Add project context so notes, memos, and downloads have the right heading.
  3. Confirm the detected fields on the left.
  4. Review QA flags before trusting report tables.
  5. Filter by matrix, location, analyte, date, or exceedance status.
  6. Export the active table, report note, review memo, AI brief, or review package.

File Support

CSV, TSV, and TXT files are the most reliable share formats and run fully in the local browser. Excel files can be uploaded when the browser can load the optional online spreadsheet parser; if not, save the worksheet as CSV and upload that file.

E-Rev looks for the most likely header row, so common lab-export title rows above the table can be skipped automatically.

Portfolio Brief

The Portfolio Brief section gives the plain-language explanation: what the project does, where it saves consulting time, how AI fits, what skills it demonstrates, and what a production build could add. The Brief button downloads a one-page HTML handout.

Consulting Value

E-Rev is designed to reduce repetitive spreadsheet review, standardize early QA checks, and make outputs easier to audit. The intent is to support consultant judgment, not replace it.

Standards Boundary

E-Rev supports preliminary data review. It does not perform formal laboratory data validation, QAPP/SAP compliance review, statistical compliance testing, risk assessment, cleanup-level selection, or regulatory closure determinations.

Use the outputs as screening-level prompts, then confirm against the project work plan, lab report, validated data package, governing agency guidance, and verified criteria library.

Expected Data Columns

Sample ID Location Sample Date Matrix Analyte Result Units Reporting Limit Qualifier Depth / From / To Latitude Longitude

Location, sample date, matrix, analyte, result, and units drive most of the review. Sample ID, reporting limits, qualifiers, depth or From/To intervals, and coordinates improve QA checks, trends, map view, and depth profiles.

Plain-text spreadsheet units such as ug/L and ug/m3 are accepted. Charts and tables display those units as µg/L and µg/m³, while CSV exports keep the plain-text unit labels.

If analytes are stored as separate result columns, map those columns as Analyte Result.

Project Setup

Project details are saved in this browser for the local app. They are used in the report note, review memo, and cleaner export filenames.

QA Flags

QA flags are spreadsheet-level review prompts, not final data validation decisions. They point to missing fields, unit conversion issues, reporting limits above criteria, rejected or blank-qualified results, negative or zero detected values, possible duplicates, and analytes without active criteria.

Formal validation may also review blanks, surrogate recovery, calibration, holding times, instrument checks, preservation, chain of custody, method-specific QC, and project-specific acceptance limits.

Review Dashboard

Dashboard cards, chart bars, and ranked lists use the active filters. They summarize exceedances, QA issues, trend status, and detection frequency for the current view.

Filters and Exports

Filters apply to the dashboard, result tables, trend choices, report note, review memo, and CSV exports. Clear filters before exporting when you need the full dataset.

When Exceedances only is active, tables narrow to exceedance records. Trend and depth views preserve the full matching series for any group with an exceedance so charts do not lose supporting time-series or vertical-profile points.

Review Memo

The memo export uses the current project setup and active filters. It includes dataset summary, QA summary, screening summary, top exceedances, trend summary, notes, and a criteria caveat.

AI Review Prompt

The AI prompt export packages the current deterministic findings into a drafting prompt. Use it to ask an AI assistant for consultant-style memo language without asking the AI to redo the math.

AI Review Assistant

The assistant panel creates a local draft brief from E-Rev findings and checks pasted AI language for missing counts, missing caveats, and common overconfident regulatory wording. It does not send data to an outside AI service.

Review Package

The review package export creates one HTML file with the current filtered summaries, tables, memo text, AI prompt, AI assistant brief, and chart snapshots for internal sharing.

Trends

Trend labels are a first-pass screen using dated detections by location, matrix, and analyte. Nondetects are excluded from the trend fit.

A trend requires at least 3 unique sample dates. When the same location, matrix, and analyte has multiple results on one date, E-Rev keeps the highest result for that date so duplicate rows or soil boring depth intervals do not create vertical time-trend lines.

Trend math uses a simple log-linear fit on positive detected concentrations only. It is not a Mann-Kendall test, Sen's slope, prediction-limit test, confidence-interval compliance test, plume model, or formal groundwater statistical evaluation.

Stable means no clear trend signal under the current screening rule: either R² is below 0.20 or the modeled change is smaller than ±15% per year. If the fit is stronger and the modeled change exceeds that threshold, positive slopes are labeled Increasing and negative slopes are labeled Decreasing.

Same-day depth profiles are not treated as time trends. Use the trend view as a screening aid and confirm conclusions with the project hydrogeologic or sampling context.

Depth Profiles

Depth profiles use records with a mapped depth field, such as 0-2 ft or 4-6 ft. Separate From and To columns are also supported. E-Rev groups them by location, date, matrix, and analyte, then plots concentration against depth.

Profiles are useful for soil borings and other same-day vertical sampling. They require at least 2 unique depth intervals.

Map View

Map View uses mapped latitude and longitude fields. The north arrow and latitude/longitude axes preserve an equal local site scale.

Basemap mode uses online map tiles when available and keeps a practical minimum window around single-location filters. The Google Maps button opens the center of the current filtered mapped locations in Google Maps.

Screening Criteria

The starter values are placeholders for testing the workflow. Replace them with verified project, state, federal, or client standards before using the output in a formal deliverable.

Screening levels are comparison values, not automatic cleanup standards. Document the source, medium, exposure assumption, units, and effective date for any criteria used in a real review.

Math guide

How E-Rev Calculates Results

Calculation Boundary

E-Rev uses deterministic spreadsheet math for first-pass screening. It does not perform formal data validation, risk assessment, groundwater statistical compliance testing, cleanup-level selection, or regulatory decision-making.

All formulas use the currently mapped fields, active criteria, and active filters. If a field is mapped incorrectly, the math will follow that mapping, so field confirmation is part of the review.

Record Parsing

Detected result

E-Rev extracts the first number from the result cell. A result is treated as nondetect when the result starts with <, contains ND, or the qualifier contains U.

Detection count

detections = records where result is numeric and not nondetect

Canonical analyte

Common aliases are grouped for matching, such as PCE = Tetrachloroethene, TCE = Trichloroethene, and Total Xylenes = Xylenes.

Unit Normalization

E-Rev normalizes results to one display unit per matrix before screening: groundwater/water to µg/L, soil/sediment to mg/kg, and air/soil gas to µg/m³.

Water

mg/L x 1000 = µg/L

ng/L x 0.001 = µg/L

Soil

µg/kg x 0.001 = mg/kg

ppm = mg/kg

Air

mg/m³ x 1000 = µg/m³

ng/m³ x 0.001 = µg/m³

Criteria Screening

Criteria match

E-Rev matches criteria by canonical analyte name and matrix. If no active criterion exists, the record is flagged as No criterion.

Exceedance

exceeds = detected normalized result > normalized criterion

Multiple

multiple = normalized result / normalized criterion

Nondetect reporting limit

If a nondetect reporting limit is greater than the criterion, E-Rev flags RL above criterion but does not count it as an exceedance.

QA Flags

QA flags are rule-based review prompts. E-Rev flags missing location, date, analyte, result, or units; unconverted units; reporting limits above criteria; detected analytes without criteria; B or R qualifiers; possible duplicate rows; negative detected results; and zero detected results.

Trend Screening

Trend screening is a simple log-linear screen, not a formal groundwater statistical test. Nondetects, zero detected values, and negative detected values are excluded from trend fitting.

Grouping

group = location + matrix + analyte

At least 3 unique sample dates are required.

Same-day rule

If a group has multiple results on one date, E-Rev keeps the highest normalized result for that date.

X value

x = years since first sample date

Years are calculated using 365.25 days.

Y value

y = ln(normalized concentration)

Regression

y = intercept + slope x

R² = 1 - SSresidual / SStotal

Change per year

percent/year = (e^slope - 1) x 100

Status

Stable if R² < 0.20 or change is smaller than ±15%/year. Otherwise positive change is Increasing and negative change is Decreasing.

Next estimate

next = e^(intercept + slope x)

The next x value uses the median spacing between prior sample dates, or 0.25 years if spacing cannot be estimated.

Depth Profiles

Depth profiles are for same-day vertical sampling, such as soil borings. They are grouped by location, date, matrix, and analyte, and require at least 2 unique depth intervals.

Depth interval

E-Rev reads depth labels such as 0-2 ft or separate From and To columns.

Midpoint

midpoint = (top + bottom) / 2

Duplicate depth

If duplicate depth midpoints exist in one profile, E-Rev keeps the highest normalized result for that depth.

Profile range

range = shallowest top depth to deepest bottom depth

Map Calculations

Map locations are grouped by sample location. E-Rev averages the latitude and longitude for each location, then projects those points to a local meter-based plot centered on the filtered locations.

Longitude to east-west meters

x = (longitude - origin longitude) x 111,320 x cos(origin latitude)

Latitude to north-south meters

y = (latitude - origin latitude) x 110,540

Status

A location status is Exceeds if any active record at that location exceeds criteria, QA review if it has QA flags, otherwise Below / nondetect.

Filtered Views

Tables narrow to the active filters. When Exceedances only is active, trend and depth charts keep the full supporting series for groups that contain an exceedance so the charts do not lose context.
